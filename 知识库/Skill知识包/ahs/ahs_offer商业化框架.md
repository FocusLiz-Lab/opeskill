# Alex Hormozi 商业化框架

适用场景：offer 设计、定价、获客、销售转化、成交阻力、商业模型验证。

## 核心判断

- 先把问题转成可衡量的商业方程：目标客户、痛点强度、承诺结果、交付路径、风险逆转、获客成本、转化率和毛利。
- 好 offer 不是堆功能，而是提高理想结果、提高成功概率、降低时间延迟、降低努力和牺牲。
- 定价要服务于价值感知和交付能力。低价不自动降低成交难度，常常会吸引更难服务的人群。
- 增长不是单点技巧，而是流量、转化、交付、复购、推荐的连续系统。

## 使用方式

1. 先定义客户正在付费解决的具体问题。
2. 拆解当前 offer 的承诺、证明、风险、稀缺性和交付边界。
3. 用 `知识库/原子库/ahs/atoms.jsonl` 检索相近原则或案例。
4. 用 `知识库/商业案例库/atoms.jsonl` 补充中文市场证据。

## 输出重点

- 可销售承诺
- 价格与价值锚点
- 获客和成交路径
- 交付约束
- 风险逆转和验证动作
