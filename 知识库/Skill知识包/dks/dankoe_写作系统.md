# Dankoe 写作系统框架

## 写作循环

```text
Observation -> Problem -> Point of view -> Mechanism -> Story/proof -> Practical takeaway -> Offer bridge
```

好内容不是资料摘要，而是带观点的公开思考。

## 角度类型

- Contrarian：大多数人以为 X，但真正问题是 Y
- Practical：我如何做 X
- Personal：我以前以为 X，后来 Y 改变了
- Diagnostic：如果你卡在 X，真正瓶颈可能是 Y
- Case-based：我/用户做了 X，结果发生了 Y
- Offer education：买/用 X 之前，先理解 Y

一篇只选一个主角度。

## 长文结构

```text
Hook
Reader problem
Personal or market observation
Main claim
Mechanism
Example or proof
Action steps
Soft offer bridge
```

## 短内容结构

```text
Sharp first line
Problem
Insight
Steps or example
One-line takeaway
Optional CTA
```

## 质量检查

- 读者是否具体
- 主张是否明确
- 前 3 行是否筛选读者
- 是否有机制而不只是鸡汤
- 是否有个人例子或案例
- CTA 是否自然
