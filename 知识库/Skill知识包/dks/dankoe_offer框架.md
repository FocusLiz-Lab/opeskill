# Dankoe Offer 框架

## Offer 定义

Offer 是一个绑定了买家、痛点、结果、机制、容器和价格的承诺。

```text
Buyer -> Pain -> Desired result -> Mechanism -> Proof -> Container -> Price -> Test
```

## 判断顺序

1. 买家是否具体
2. 痛点是否紧急
3. 忽视痛点是否有成本
4. 用户是否已经尝试解决
5. 交付者是否能在 30 天内制造可见结果

少于 3 条成立，不要急着做产品。

## 常见产品容器

- Done-for-you service：最快验证现金和证明
- Done-with-you consulting/coaching：适合诊断型问题
- Digital product：适合已有受众和重复方法
- Template/toolkit：适合结构化执行
- Subscription/community：适合持续身份、陪伴、策展

新手优先服务或咨询，不要一上来做课。
