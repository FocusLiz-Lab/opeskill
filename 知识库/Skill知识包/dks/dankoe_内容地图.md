# Dankoe 内容地图框架

## 内容不是随机发帖

内容是公开思考系统：

```text
Problem -> Perspective -> Proof -> Practical lesson -> Offer bridge
```

## 内容定位

使用这个句式：

```text
I explore {domain} for {audience} who want {result}, from the point of view that {strong belief}.
```

不要只写分类词，比如 AI、成长、赚钱、自媒体。

## 内容地图四层

- Core problem：用户真正想解决什么
- Belief shift：用户误解了什么
- Mechanism：结果实际如何发生
- Proof/story：个人经历、案例、实验、失败、用户反馈

## 内容支柱

可从这些类型中选 3-5 个：

- Problem education
- Personal transformation
- Tactical how-to
- Opinion/worldview
- Proof/case/build-in-public
- Offer education

每个支柱必须配示例选题。

## 长转短系统

1. 每周一篇长文/Newsletter/脚本
2. 拆 5-10 条短内容
3. 把评论和异议变成下一篇主题
4. 部分内容自然桥接 offer
