# Dankoe AI 工作流框架

## 核心原则

AI 不是业务。AI 是加速器。

现代一人公司仍然需要：

```text
Brand -> Content -> Offer -> Sales page -> Traffic -> Feedback -> Iteration
```

AI 应该进入其中一个具体环节，而不是制造复杂但没有买家信号的流程。

## 先选一个业务循环

- Research loop
- Content loop
- Offer validation loop
- Product creation loop
- Client delivery loop
- Learning loop
- Weekly review loop

不要一次设计端到端巨型系统。

## 判断必须由人负责

这些地方不能完全交给 AI：

- audience selection
- taste and angle
- offer promise
- final publishing
- pricing
- diagnosis
- feedback priority

## 工作流字段

- Inputs
- AI tasks
- Human tasks
- Output artifact
- Review metric
- Cadence

## Prompt Stack

- Context prompt：用户身份、受众、offer、观点
- Research prompt：提取问题、异议、案例、hook
- Creation prompt：生成草稿、offer、页面、资产
- Critique prompt：找泛、弱、虚、无证明
- Review prompt：总结信号和下一轮动作
